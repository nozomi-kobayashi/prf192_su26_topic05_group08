## PC Component Sales & Compatibility Validation System (PRF192 - Topic 05)

### 📌 Project Overview
This repository contains a structured C application designed for managing PC component inventory and automating the validation process for PC assembly orders based on custom hardware compatibility rules.

### 🛠️ Core Technical Requirements
- **Language:** C (Standard Structured Programming).
- **Architecture:** Modular programming with decoupled `.h` (interfaces) and `.c` (implementations) files.
- **Memory Management:** Dynamic arrays utilizing `malloc`, `realloc`, and `free` to guarantee memory safety and zero leaks.
- **Data Persistence:** Automatic data loading on startup and saving upon exit via file I/O operations (Text/Binary).
- **Validation:** Strict error code handling using conditional programming (No exceptions handled).

### 🚀 Folder Structure
- `/src`: Contains all source code files (`main.c`, `component.c`, `assembly.c`, `compatibility.c`).
- `/include`: Contains header files (`component.h`, `assembly.h`, `compatibility.h`).
- `/data`: Storage for input/output text or binary data files.
- `/docs`: Contains Technical Report (PDF), Flow-charts, and Presentation slides.
